> Context7-sourced snapshot (2026-07-23) for the `post-session-report` (S-04) analysis route.
> Source library: `/vercel/next.js/v16.2.9` (matches installed `next@16.2.9`).
> Companion to `research.md` §4 and §10. Artifacts in English (project convention).
>
> This app is **Next.js 16** — request interception lives in `src/proxy.ts`, not middleware.
> Reuse the existing API pattern from `src/app/api/realtime/token/route.ts`
> (auth via `createClient()` + `getUser()`, `OpenAI-Safety-Identifier` = sha256(user.id),
> 401 on no user, 502 on upstream failure, `Cache-Control: no-store`).

## 1. Route segment config (what a `route.ts` can export)

```typescript
export type AppSegmentConfig = {
  revalidate?: number | false
  dynamicParams?: boolean
  dynamic?: 'auto' | 'error' | 'force-static' | 'force-dynamic'
  fetchCache?:
    | 'auto' | 'default-cache' | 'default-no-store'
    | 'force-cache' | 'force-no-store' | 'only-cache' | 'only-no-store'
  runtime?: 'edge' | 'nodejs'
  maxDuration?: number
}
```
Source: https://github.com/vercel/next.js (app-route module userland shape)

For `/api/report`:
- `export const maxDuration = 30` — headroom for the model call (+ optional retry). Well under the
  Vercel function cap (research.md notes the 300 s Hobby cap applies to this analysis route).
- `runtime` — leave default (`nodejs`); the OpenAI call and Supabase auth need Node, not edge.
- No caching config needed: **POST handlers are always dynamic**; only `GET` is statically
  optimizable and only if it opts in (see §4).

## 2. POST handler skeleton (mirrors the token route)

```typescript
import { NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'

export const maxDuration = 30

export async function POST(request: Request) {
  // Auth: defense-in-depth on top of src/proxy.ts (which already 401s /api/*)
  const supabase = await createClient()
  const { data: { user }, error } = await supabase.auth.getUser()
  if (error && error.name !== 'AuthSessionMissingError') {
    console.error('report getUser failed:', error.message)
  }
  if (!user) {
    return NextResponse.json({ error: 'unauthorized' }, { status: 401 })
  }

  const { transcript } = (await request.json()) as { transcript: string }

  // ... call OpenAI Responses API (see docs-openai-responses-structured-outputs.md) ...
  // wrap the upstream call in try/catch → 502 { error: 'upstream_error' } on failure,
  // reusing the token route's error shape.

  return NextResponse.json(report, {
    headers: { 'Cache-Control': 'no-store' },
  })
}
```

## 3. `after()` — only if persistence is added later (S-05 territory)

If a report is ever persisted without making the user wait, use `after()` from `next/server`
(stable in Next 16). It runs the callback **after the response is sent** and does not make the
route dynamic. Return the report to the client first, persist in `after()`.

```ts

import { cookies, headers } from 'next/headers'

export async function POST(request: Request) {
  // ... produce `report` ...

  after(async () => {

    const sessionCookie = (await cookies()).get('session-id')?.value || 'anonymous'

  })

  return new Response(JSON.stringify(report), {
    status: 200,
    headers: { 'Content-Type': 'application/json', 'Cache-Control': 'no-store' },
  })
}
```
Source: https://github.com/vercel/next.js/blob/v16.2.9/docs/01-app/03-api-reference/04-functions/after.mdx

`after()` respects the route's `maxDuration` (platform default or configured).
Source: same after.mdx ("Reference > Duration").

**S-04 recommendation**: keep it a synchronous POST that returns the report JSON — no `after()`,
no persistence. Persistence/archive (FR-015) is a later slice per the roadmap (first migration = S-05).

## 4. Caching note (background knowledge)

GET route handlers are **not** statically generated by default (Next 15+); they run dynamically
unless they export `dynamic = 'force-static' | 'error'`, `revalidate`, or `generateStaticParams`.
POST is never cached. So the report route needs no explicit cache opt-out for correctness — the
`Cache-Control: no-store` header is belt-and-suspenders, matching the token route.
Source: https://github.com/vercel/next.js (is-static-gen-enabled) and
https://github.com/vercel/next.js/blob/v16.2.9/docs/01-app/02-guides/upgrading/version-15.mdx

## 5. Proxy interaction (do not break the session gate)

`src/proxy.ts` matches and gates `/api/*`. Its matcher excludes image-extension paths — so **do not**
name a report route with an image suffix (e.g. `/report.png`), which would bypass the gate.
Plain `/api/report` (JSON) is matched and gated. See `research.md` §4.
